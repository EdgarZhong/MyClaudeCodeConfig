## 任务一：整理 `docs/` 目录

`docs/` 下目前有大量历史文档（各阶段的实施手册、排查文档、调优报告等）。这些文档大多写于早期开发阶段（阶段零到阶段五），其中的参数设定、架构描述、实验方向等信息可能已经过时。

**操作：**

1. 在 `docs/` 下新建 `research/` 子目录，放入上述三份最新研究文档
2. 在 `docs/` 下新建 `archive/` 子目录（如果还没有的话）
3. 逐一检查 `docs/` 下现有的文档，对照三份研究文档中的最新信息，判断哪些已经过时。过时的文档移入 `docs/archive/`，**不要删除任何文件**
4. 判断标准：如果一份文档中的核心内容（参数、架构、实验方向）已经被研究备忘录 v7 或点子库 v4 取代或否定，就算过时

### 关于参数约定的处理

旧 README 中有大量训练和评测参数写死在文档里，但这些参数会随着实验阶段推进而变化（比如当前阶段的统一超参见研究备忘录 v7 第 5.2 节，和之前的阶段不同）。**请思考一个更好的参数管理方式**——参数约定是否应该放在 README 中，还是放在专门的配置文件或文档中，由各阶段的实验设计来维护？你来做这个工程决策。

### 不要包含的内容

- 阶段零到阶段五的开发过程叙事（那是历史）
- 环境搭建的完整步骤（conda create、pip install 等）——只保留激活命令
- 旧模型在白边原图上的结果
- 数据集清洗的历史过程
- 常见问题与说明（如果仍然有用，单独放一个文档）

### 必须遵守的规则

- 遵循你的全局 Claude 规则（`CLAUDE.md` 或全局配置中的项目规范）
